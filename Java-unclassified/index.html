面向对象编程的本质是：以类的方式组织代码，以对象的组织封装数据
A a = new A (); 其中 a 变量称作引用变量，其类型为 A，指向了以 A 类为模板，在堆空间里创建的 A 类对象。
构造方法（构造器，构造函数）快捷键 alt+ins，使用 alt+shift 选中多行：1。和类名相同 2、无返回值 作用：1.new 本质在调用构造方法 2. 初始化对象的值 注：定义有参构造后想使用无参构造就必须显式定义
封装：高内聚低耦合 - 高内聚：类的内部数据操作细节自己完成，不允许外部干涉；低耦合：仅暴露少量方法给外部使用 1. 提高程序安全性保护数据 2. 隐藏代码实现细节 3. 统一接口 4. 增加系统可维护性
继承 crtl+h 打开族谱：子类可以可以通过继承获取父类原有的字段和方法，也可以在父类的基础进行扩展，使用 extends 来表示子类和父类之间的继承关系，java 的类之间只支持单继承，一个类只能有一个直接父类，可以多重继承
super：1. 子类构造方法中第一行隐藏地调用了父类的无参构造，想显式调用父类的构造方法必须在子类的第一行 2.super 必须只能出现在子类的方法或构造方法中 3.super 和 this 不能同时调用构造方法
final 修饰后无法被继承
方法重写 alt+ins 选择 override：即 b 是 A new 出来的对象，因此调用了 A 的方法，因为静态方法是类的方法，而非静态是对象的方法，有 static 时，b 调用了 B 类的方法，因为 b 是用 b 类定义的，没有 static 时，b 调用的是对象的方法，而 b 是用 A 类 new 的
重写（子类的方法和父类必须一致，方法体不同）：需要有继承关系，子类重写父类的方法后子类对象调用时一定会调用子类重写后的方法而不会调用父类。1. 方法名必须相同 2. 参数列表必须相同 3. 修饰符：范围可以扩大不能缩小 4. 异常：范围可以缩小不能扩大
为什么需要重写：父类的功能子类不一定满足
多态：
向上转型和向下转型：自动类型转换和强制类型转换是在基本数据类型方面的，向上向下转型是引用类型转换
向上转型：子 -> 父，java 中自动转换 父类型的引用指向子类型的对象
向下转型：父 -> 子，需要加强制转换符 想访问子类独有的方法
为什么叫多态？
多种形态，编译的时候一种形态运行的时候另一种形态。
多态是什么？
父类型的引用指向子类型的对象
java 程序分为编译阶段和运行阶段：1. 编译阶段：编译器只知道该引用变量的类型是父类型，所以会去父类找调用的方法并与之绑定，编译阶段的绑定属于静态绑定；2. 运行阶段：实际上在堆内存中创建的是此引用变量指向类的对象，即子类对象，所以在运行阶段会动态执行子类对象的方法，这个过程属于运行阶段绑定，运行阶段的绑定属于动态绑定。
多态怎么做？（多态性）：这样，不用修改源程序代码，就可以让引用变量绑定到各种不同的类实现上，从而导致该引用调用的具体方法随之改变，即不修改程序代码就可以改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

HashMap 集合（有书签）：1.HashMap 集合底层是哈希表 / 散列表数据结构 2. 哈希表是怎样的数据结构？是一个一维数组，数组中每个元素是一个单链表，数组查询效率高，单链表随即增删效率高，哈希表将两种结合充分发挥其优点
HashMap 的哈希算法：return (key == null) ? 0 : (h = key.hashCode ()) ^ (h >>> 16); 代码解释：key 为空时得到哈希值为 0，不空时先根据 key 对象调用其类重写的 hashCode () 方法并赋值给 h，再和右移 16 位的 h 做异或运算。简单解释：（jdk8）通过 hashCode () 的高 16 位异或低 16 位实现
1. 为什么这样实现？主要是从速度，功效和质量来考虑的，减少系统的开销，也不会造成因为高位没有参与下标的计算从而引起的碰撞 2. 为什么要用异或运算符？保证了对象的 hashCode 的 32 位值只要有一位发生改变，整个 hash () 返回值就会改变。尽可能的减少碰撞。
（putVal 方法内）使用哈希算法后计算数组下标：int index = hash & (arrays.length-1);
为什么 HashMap 的数组长度是 2 的整数幂？2 的 n 次方 - 1 的二进制数全为 1，在进行位与运算时能在每一位上得到 0 或 1，增加了随机性，保证了空间的使用效率，若有一位是 0 时，那么结果的这一位只能是 0，随机性降低了
hashmap.put (k,v) 实现原理（有书签 https://www.cnblogs.com/yanzige/p/8392142.html）：调用哈希函数获取 Key 对应的 hash 值，再计算其数组下标；如果没有出现哈希冲突，则直接放入数组；如果出现哈希冲突，则以链表的方式放在链表后面；如果链表长度大于 8 且数组长度大于 64 转成红黑树，满足链表长度不满足数组长度是扩容，链表长度低于 6，就把红黑树转回链表；
如果结点的 key 已经存在，则替换其 value 即可；若存放总数大于阈值（默认为 12：默认数组长度 16，负载因子 0.75，阈值 = 数组长度负载因子），调用 resize 方法进行数组扩容。（具体点的在书签的代码注释的流程判断）
hashmap.get (k,v) 实现原理（有书签 https://blog.csdn.net/llllllkkkkkooooo/article/details/114702589）：1. hash 方法算 hash 值（利用 key 的 hashCode 进行高 16 位异或低十六位运算） 2. 调用 getNode () 方法如果数组为 null 则直接返回 null 3. 如果数组不为 null，则进行位运算形式的取模运算得到索引位置 4. 比较索引位置的第一个元素的 hash 值与 key 值是否都相同
（源码：first.hash == hash && ((k = first.key) == key || (key != null && key.equals (k)))) 其中 key 先用比较内存地址，若是两个引用变量相等即均指向同一个内存地址时，它们的内容也一定相等，再用或运算连接 equals 方法比较内容，这样做的好处是只比较内存地址，equals 会依次比较这个对象的每个属性内容，若前者为 true 则无需比较后者，效率更快）
5. 满足的话直接返回该节点，不满足的话先看是不是红黑树，是的话就调用红黑树的 get 方法，不是的话则遍历链表比较，如果最后一个节点满足的话就返回，如果还不满足就返回 null

自定义类型可排序吗？无法排序，因为没指定比较规则，报 java.lang.ClassCastException，没实现 java.lang.Comparable 接口。有两种方式，法一：需要实现此接口的 compareTo 方法（返回值 > 0 代表在右子树，<0 在左，=0 值相同，覆盖） 法二，构造 TreeSet/TreeMap 集合时传参，参数是实现 java.util.Comparator 接口的对象（其中必须实现它的 compare 方法）
Comparable 和 Comparator 怎么选？若比较规则不会变化或只有一个时，选择前者较好；若有多个且需要多个规则间频繁切换，则后者较好

File 类方法（百度）
序列化（Serialize）：将 java 对象的状态保存下来存储在文件中的过程。使用 ObjectOutputStream
反序列化（DeSerialize）：将硬盘上的数据重新恢复到内存中，恢复成 java 对象。使用 ObjectInputStream
1. 参与序列化和反序列化的对象必须实现 Serializable 接口，否则报 java.io.NotSerializableException
2. 多个对象进行序列化可使用集合存储，将集合序列号。注：不仅对象的类需要实现 Serializable 接口，集合也需要实现
3.transient 关键字表示游离的，被修饰的属性不会参与到序列号中。注：(1).transient 关键字只能修饰变量，而不能修饰方法和类；(2). 一个静态变量不管是否被 transient 修饰，均不能被序列化；
(3). 被 transient 关键字修饰的变量真的不能被序列化吗？不一定，若实现的是 Serializable 接口，则所有的序列化将会自动进行，若实现的是 Externalizable 接口，则没有任何东西可以自动序列化，需要在 writeExternal 方法中进行手工指定所要序列化的变量，这与是否被 transient 修饰无关。来源：https://www.runoob.com/w3cnote/java-transient-keywords.html
4.Serializable 接口只是一个标志接口，（接口当中什么代码也没有）起到标识的作用，jvm 看到这种接口会自动生成一个序列化版本号 serialVersionUID，建议显式声明不建议自动生成 eg：private static final long serialVersionUID = 1l;
注：(1). 在反序列化时 jvm 识别一个类先通过类名，若类名一致，再识别序列化版本号，若号不一致会抛出 java.io.InvalidClassException
(2). 自动生成序列化版本号有什么缺陷？一旦代码确定就不能进行后续修改，因为只要修改就会重新编译，会生成新的号，jvm 认为这是全新的类，所以凡是实现 Serializable 接口的类建议要显式声明固定的序列化版本号
properties 与 io：新建 properties 集合对象 pro，pro.load (reader) 方法传入一个输入流（字节流字符流均可）对象即可将文件中的数据顺着管道加载到 Map 集合，此类文件一般为配置文件，后缀一般是 properties

多线程：
进程与线程：进程（Process）是资源分配的基本单位，线程（Thread）是 CPU 调度的基本单位。
进程：是程序在某个数据集合上的一次运行活动，也是操作系统进行资源分配和保护的基本单位。进程就是程序的一次执行过程，程序是静态的，它作为系统中的一种资源是永远存在的。而进程是动态的，它是动态的产生，变化和消亡的，拥有其自己的生命周期。
线程：进程中的一个执行任务（控制单元），负责当前进程中程序的执行。一个进程至少有一个线程，一个进程可以运行多个线程，多个线程可共享数据。与进程不同的是同类的多个线程共享进程的堆和方法区资源，但每个线程有自己的程序计数器、虚拟机栈和本地方法栈，所以系统在产生一个线程，或是在各个线程之间作切换工作时，负担要比进程小得多，也正因为如此，线程也被称为轻量级进程。
进程与线程的区别：
（1）调度：线程作为调度和分配的基本单位，进程作为拥有资源的基本单位
（2）并发性：不仅进程之间可以并发执行，同一个进程的多个线程之间也可并发执行
（3）拥有资源：进程是拥有资源的一个独立单位，线程不拥有系统资源，但可以访问隶属于进程的资源.
（4）系统开销：在创建或撤消进程时，由于系统都要为之分配和回收资源，导致系统的开销明显大于创建或撤消线程时的开销。
如何实现线程？
1. 新建线程类，继承 java.lang.Thread 类，并重写 run 方法；
2.(推荐）新建线程类，实现 java.lang.Runnable 接口，实现 run 方法，再实例化此类并作为参数传入 Thread 的有参构造 Thread (Runnable target)；
继承 Thread 和实现 Runnable 接口的区别：
a. 实现 Runnable 接口避免多继承局限
b. 实现 Runnable () 可以更好的体现共享的概念
3.（可获取线程的返回值）通过 JUC 包（java.util.concurrent）下的 FutureTask 构造方法实现 Callable 接口创建线程
4.（创建多个线程时推荐）通过工厂类创建线程池（4 个方法：newCachedThreadPool ()，newFixedThreadPool (int nThreads)，newScheduledThreadPool (int corePoolSize)，newSingleThreadExecutor ()
start () 方法：启动一个分支线程，在 jvm 开辟一个新的栈空间，只要新空间开启方法就结束，线程随即启动成功，启动成功的线程会自动调用 run 方法，且 run 方法在分支栈的栈底（main 方法在主栈底，run 和 main 是平级的）
线程生命周期（书签）
修改线程名字：线程对象.setName ()，获取线程名字：.getName ()，获取当前线程：Thread.currentThread ()，写在 main 方法就是 main 线程，写在 run 方法就是哪个线程执行 run 方法是
sleep () 方法 static void sleep (long millis)：
作用：让当前线程进入阻塞状态，放弃占有 cpu 时间片，当 sleep () 状态超时线程重新转入就绪状态。面试题：main 方法中 Thread t = new MyThread ();t.start ();t.sleep (1000) 会使 t 线程睡眠吗？不会。sleep 方法是静态方法，t 可以调用但与 t 对象没关系，因为出现在 main 方法，只会让当前线程也就是 main 线程睡眠
如何中断睡眠？t.interrupt ()（此方式依靠 java 异常处理机制 InterruptException）另：run 方法当中异常不能 throws，只能 trycatch，因为 run 方法在父类没抛出任何异常，子类不能比父类抛出更多异常 3
终止线程执行：stop 方法终止会直接 kill 掉线程，还没保存的数据会丢失，所以建议使用一个默认 true 的 boolean 标记做判断，if 标记 true 执行线程，else 中保存各类数据后直接 return，需要终止时将标记赋为 false
线程调度方法：
static void yield () 让位方法，暂停当前正在执行的线程并执行其他线程，让当前线程从运行状态回到就绪状态。注：回到就绪后有可能还会再次抢到
void setPriority (int p) 设置线程优先级 优先级 1~10，默认 5，越高的获取 cpu 时间片 可能 会多一些（大概率方向上更偏向高的）
int getPriority () 获取线程优先级
void join () 合并线程，使当前线程进入阻塞，调用此方法的线程执行，直到这个线程结束当前线程才能执行
什么时候数据在多线程并发环境下会存在安全问题？三个条件：1. 多线程并发；2. 有共享数据；3. 共享数据有修改行为。
如何解决线程安全问题？线程同步。目前主要有三种线程安全实现方法：互斥同步（阻塞同步）、非阻塞同步以及无需同步的线程安全方案。
互斥同步是指在多个线程并发访问共享数据时，保证共享数据在同一时刻只被一个（或一些，使用信号量的时候）线程使用。简单的说就是线程排队执行。
异步编程模型：在进程中任务未结束但在等待的过程中可先去执行下一个任务（多线程并发）
同步编程模型：在进程中任务未结束时需等待结束才能执行下一个任务 （线程排队执行）
线程同步（更多内容在书签 https://zhuanlan.zhihu.com/p/377423211）：
・在 Java 中所有的对象，都会有一把锁，叫做内置锁，也称作监视器锁。这是种排他锁。（排他锁：一个线程获取后，其他线程只能等待其释放后，才有机会获得该锁。）
・synchronized 是 Java 中的关键字，依赖 JVM 实现，是一种同步锁。它修饰的对象有以下几种：
　　 1. 修饰一个代码块，被修饰的代码块称为同步语句块，其作用的范围是大括号 {} 括起来的代码，作用的对象是小括号内的对象（一般是 this 即调用这个代码块的对象）；注：这个对象只能是共享对象，不能是局部变量，如果是方法体中的局部变量会导致不同线程 new 出的该变量不是同一对象
　　 2. 修饰一个方法，被修饰的方法称为同步方法，其作用的范围是整个方法，作用的对象是调用这个方法的对象；
　　 3. 修饰一个静态的方法，其作用的范围是整个静态方法，作用的对象是这个类的所有对象；
　　 4. 修饰一个类，其作用的范围是 synchronized 后面括号括起来的部分，作用的对象是这个类的所有对象。
・线程同步执行原理：1. 假设线程 1 先执行，遇到 synchronized，就会自动找 synchronized 作用的共享对象的锁并占有，之后执行作用范围的代码，执行过程中持续占有锁直至作用范围的代码运行完毕，锁才会释放；
2. 假设线程 1 占有锁时，线程 2 也遇到 synchronized，尝试占有这把锁时发现被占有，进入同步阻塞状态，直到线程 1 结束这段代码并归还锁后，线程 2 才会去竞争锁（jdk1.6 前）。
守护线程：java 线程分为两类：用户线程和守护线程（后台线程），代表性：垃圾回收线程是守护线程。注：主线程 main 方法是一个用户线程
特点：一般是死循环，持续运作，所有用户线程只要结束守护线程自动结束。eg：使用定时器和守护线程，每到 0 点自动备份
用法：（线程对象）t.setDaemon (true)
定时器：实现方式：1.（最原始）sleep 设置睡眠时间；2.（很少用）java.util.Timer；3.（实际开发用最多）Spring 框架中的 SpringTask 框架
Callable：实现方式：FutureTask task = new FutureTask (new Callable () {重写 call 方法}); Thread t = new Thread (task); 注：FutureTask 类实现了 RunnableFuture 接口，而此接口继承了 Runnable 接口，所以可以作为参数传入 Thread 构造方法
如何在其他线程获取返回值？task.get ()，此方法在执行时若返回值线程还未完成任务，则会使获取返回值的线程阻塞并进入等待队列的队列头；若返回值线程完成任务但还没保存到 outcome 返回值字段（state == COMPLETING 状态），则只需 Thread.yield () 让出 cpu 即可。注：详细源码看 FutureTask 的 awaitDone 方法
优缺点：优：可获取线程执行结果 缺：效率较低，获取执行结果时有可能时当前线程受阻塞
wait 和 notify 方法（生产者和消费者模式）：Object 类自带。eg：Object o = new Object ();o.wait () 使正在 Object 对象上活动的线程进入无期限等待状态，并且释放之前占有的 o 对象的锁，直至调用 o.notify () 方法唤醒（ 随机 唤醒在 o 上等待的线程且不会释放锁）。
notifyAll () 方法：唤醒 o 对象上处于等待的 所有 线程
